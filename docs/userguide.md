# HTR Polygon Annotation Tool (Yat) - Руководство пользователя

HTR Polygon Annotation Tool (Yat) — это веб-приложение для подготовки и аннотирования данных для распознавания рукописного текста (HTR/OCR). Приложение позволяет выполнять полный цикл подготовки данных: от кадрирования оригинальных сканов до точной полигональной сегментации текстовых строк и ввода текста.

## Создание проекта

1. На главной странице нажмите **«+ Создать проект»**
2. Введите название и описание проекта
3. Нажмите **«Создать»**

Проект появится в списке на главной странице.

---

## Загрузка изображений

1. Откройте страницу проекта
2. Перетащите изображения в область **«Перетащите изображения сюда или нажмите для выбора»**
3. Или нажмите на область и выберите файлы (JPG, PNG)

Изображения появятся в списке со статусом **«Нужно кадрировать»**.

---

## Кадрирование (Cropper)

1. На странице проекта нажмите кнопку **«Обрезка»** у изображения
2. Выделите область кадрирования (drag mouse)
3. При необходимости поверните рамку за маркер вращения
4. Нажмите **«✂️ Сохранить»**

После обрезки статус изображения изменится на **«Кадрировано»**.

> **Недеструктивность:** Оригиналы сохраняются в `data/originals/`, поэтому можно перекадрировать изображение в любой момент.

---

## Сегментация (Editor)

1. На странице проекта нажмите кнопку **«Сегменты»** у изображения
2. Нажмите **«Рисовать (N)»** или клавишу **N** для включения режима рисования
3. Кликните 4-5 раз для создания полигона вокруг строки текста
4. Нажмите **«Выделение (V)»** или клавишу **V** для режима выделения
5. Перетащите вершины для точной подгонки

Автосохранение работает автоматически при изменении полигонов.

---

## Распознавание текста (Text Editor)

1. На странице проекта нажмите кнопку **«Распознавать»** у изображения
2. Кликните на полигон на левом холсте
3. Отредактируйте текст в поле справа
4. Нажмите **«Сохранить и следующий (Enter)»** или закройте модальное окно

## Пакетная обработка

### Пакетная детекция (YOLOv9)

1. На странице проекта в разделе **«Пакетная обработка»** нажмите **«Запустить детекцию полигонов»**
2. Задача будет создана и запущена в фоновом режиме
3. Прогресс можно отследить по адресу `/api/tasks`
4. После завершения откройте Editor — полигоны будут созданы автоматически

### Пакетное распознавание (TROCR)

1. На странице проекта в разделе **«Пакетная обработка»** нажмите **«Запустить распознавание текста»**
2. Задача будет создана и запущена в фоновом режиме
3. После завершения откройте Text Editor — текст будет распознан

## Импорт и экспорт

### Формат данных

Приложение использует формат PAGE XML для хранения аннотаций. Этот формат совместим с eScriptorium и другими инструментами для аннотации документов.

### Структура ZIP-архива для импорта

```
archive.zip
├── image1.jpg
├── image2.png
├── image1.xml      # PAGE XML аннотация
├── image2.xml
└── project.json    # Метаданные (опционально)
```

### Импорт ZIP (eScriptorium)

1. На главной странице нажмите **«Импорт»**
2. Выберите ZIP-архив с изображениями и XML-файлами (METS + PageXML)
3. Укажите коэффициент упрощения полигонов (например, 5)
4. Нажмите **«Импортировать»**

Будет создан новый проект с загруженными изображениями и аннотациями.

### Экспорт ZIP

1. Откройте страницу проекта
2. В верхней панели нажмите **«Экспорт»**
3. Браузер загрузит архив с именем `<project_name>_export.zip`

**Содержимое архива:**
- Изображения в корне архива
- XML-файлы аннотаций в формате PAGE XML
- Файл `project.json` с метаданными проекта

## Формат хранения данных

Данные хранятся в SQLite базе данных (`database.db`):

| Таблица | Описание |
|---------|----------|
| `projects` | Проекты (название, описание, дата создания) |
| `images` | Изображения (проект, статус, crop_params) |
| `annotations` | Аннотации (полигоны + распознанный текст в JSON-поле) |
| `tasks` | Фоновые задачи (детекция, распознавание, прогресс) |

### Структура аннотации (JSON в БД)

```json
{
  "regions": [
    {
      "points": [[100, 100], [200, 100], [200, 150], [100, 150]]
    }
  ],
  "texts": {
    "0": "Распознанный текст для региона 0",
    "1": "Распознанный текст для региона 1"
  },
  "status": "segment"
}
```

**Поля:**
- `regions` — массив полигонов (каждый с `points: [[x1,y1], [x2,y2], ...]`)
- `texts` — словарь `{index_региона: текст}`
- `status` — статус изображения (`"cropped"`, `"segment"`, `"texted"`)

## Горячие клавиши

### Редактор разметки (Segment)

| Клавиша | Действие |
| --- | --- |
| **N** | Режим рисования (Draw) |
| **V** | Режим выделения (Select/Edit) |
| **P** | Режим правки точек (Edit Points) |
| **Del** | Удалить выделенное |
| **Esc** | Отмена рисования / Сброс выделения |
| **Ctrl + Z** | Отменить действие (Undo) |
| **Ctrl + Shift + Z** | Вернуть действие (Redo) |
| **Ctrl + A** | Выделить все полигоны |
| **Ctrl + ← / →** | Предыдущее / Следующее изображение |
| **ПКМ** (или Alt+Drag) | Перемещение холста (Pan) |
| **Колесо мыши** | Зум (Zoom) |

### Редактор текста (Text Editor)

| Клавиша | Действие |
| --- | --- |
| **Enter** | Сохранить текст и перейти к следующему региону |
| **Ctrl + ← / →** | Предыдущее / Следующее изображение |

### Кадрирование (Cropper)

| Клавиша | Действие |
| --- | --- |
| **← / →** | Предыдущее / Следующее изображение |
| **Esc** | Отмена выделения / Возврат в проект |
